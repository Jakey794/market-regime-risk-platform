# Project Instructions

- This is a risk/regime research platform, not a stock prediction app.
- Prefer pure functions with typed signatures.
- Keep data loading separate from risk math.
- Keep dashboard code separate from portfolio/risk modules.
- All financial metric functions need deterministic unit tests.
- Backtest/regime work later must avoid look-ahead bias.
- Error messages should be clear and user-facing where appropriate.
- Use pandas/NumPy/scikit-style conventions.
- Run `ruff` and `pytest` before finishing any task.
